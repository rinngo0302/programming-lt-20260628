# シーン仕様

3シーン構成。シーン遷移の全体像は [00-overview.md](00-overview.md) を参照。

## タイトルシーン

### 役割

ゲームの入口。スタート操作を受け付けてゲームシーンに遷移する。

### 画面構成(v1)

- タイトルロゴ/タイトルテキスト
- 「スタート」ボタン(または何かキーを押すとスタート、UIボタンどちらでもよいが、まずはボタン1つのシンプルな構成とする)

### 遷移条件

- 「スタート」操作 → ゲームシーンへ遷移

### v1で扱わないもの

- キャラクター/カート選択
- コース選択(コースが1つのため不要)
- オプション/設定画面

## ゲームシーン

### 役割

レース本体。プレイヤー1台 + CPU3台が3ラップを走り、順位が決まったらリザルトシーンに遷移する。

### 画面構成(v1)

- レース画面(3Dビュー、三人称追従カメラ)
- HUD: 現在の順位、現在のラップ数 / 総ラップ数(例: `2 / 3`)、所持アイテム表示
- レース開始時のカウントダウン(3, 2, 1, GO!)

### 遷移条件

- 全レーサーがゴールした、またはプレイヤーがゴールした時点でレース終了とみなし、リザルトシーンへ遷移する(詳細な終了条件は [02-race-rules.md](02-race-rules.md) を参照)

### 責務分割の方針(MVP)

- レースの状態(順位・ラップ・所持アイテムなど)は Model 層が保持し、`ReactiveProperty` 等で外部に公開する
- HUDの各Presenterは Model の `CreateBindings()` で生成された Bindings struct を受け取り、表示のみを担当する(`CLAUDE.md`のBindings struct方針に準拠)

## リザルトシーン

### 役割

レース結果を表示し、次のアクション(再戦 or タイトルへ戻る)を選択させる。

### 画面構成(v1)

- 最終順位表(1位〜4位、プレイヤー/CPUの名前と順位)
- プレイヤーの最終順位を強調表示
- 「もう一度プレイ」ボタン → ゲームシーンへ遷移
- 「タイトルへ戻る」ボタン → タイトルシーンへ遷移

### 遷移条件

- 「もう一度プレイ」 → ゲームシーン(同じコースで再戦)
- 「タイトルへ戻る」 → タイトルシーン

### v1で扱わないもの

- タイムの記録・ベストタイム表示(順位のみで十分とする。将来拡張候補)
