# CPU AI仕様

v1のCPUは3体。複雑な戦略は持たず、シンプルなウェイポイント追従 + アイテムのランダム使用とする。

## 走行ロジック: ウェイポイント追従

- コース上に一定間隔でウェイポイント(目標地点の列)を配置する(コース定義の一部として [02-race-rules.md](02-race-rules.md) のチェックポイントとは別に、より細かい間隔で持つことを想定。チェックポイントをそのままウェイポイントとして使うかは実装時に判断してよい)
- CPUは「現在向かうべきウェイポイント」への方向にステアリングを入力し、基本的に常時加速する
- 現在のウェイポイントに十分近づいたら、次のウェイポイントを目標に切り替える
- 他のレーサー(プレイヤー/他CPU)との衝突を避ける高度な回避行動は**v1では実装しない**(簡易な進行方向の補正のみ、もしくは衝突は許容し物理的に弾かれるだけでよい)

## アイテム使用ロジック

- CPUはアイテムを取得したら、すぐには使わず、保持してからランダムな短い遅延(目安: 0.5〜2秒のランダム)後に使用する
- 使用条件の判定(例: 前方に相手がいる時だけ甲羅を撃つ、など)は**v1では行わない**。取得したらランダムなタイミングで使う、というシンプルな挙動に留める

## 難易度

- v1では全CPUの強さ(速度・反応速度)は同一とする(差をつけない)
- 将来的に「CPUごとに速度パラメータを変える」などで難易度差をつけることは拡張候補とする

## アーキテクチャメモ

- CPUの走行・アイテム使用ロジックは `CpuDriverModel`(仮)のような純粋なC#クラスに分離し、MonoBehaviour側はそれを駆動するだけにする(`CLAUDE.md`のテスト方針: ビジネスロジックをテスト容易にするため)
- ウェイポイント列もコース設定(`CourseData` ScriptableObject)の一部として保持する
