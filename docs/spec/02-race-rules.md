# レースルール

## 基本設定

| 項目 | 値 |
|---|---|
| コース数 | 1 |
| レーサー数 | 4台(プレイヤー1 + CPU3) |
| ラップ数 | 3 |
| ドリフト | なし |

## コース構造

- コースはループ状(スタート/ゴールが同一地点)
- コース上に**チェックポイント**を複数配置する(最低でもスタート/ゴールラインを含め3〜4箇所程度を想定)
- 各レーサーは「現在の周回数」と「直近に通過したチェックポイントの番号」を保持する

### チェックポイント・ラップ判定

- チェックポイントは順番に通過する必要がある(通過済みチェックポイント番号より1つ先のチェックポイントのみ通過と認める。ショートカットによる不正なラップ加算を防ぐため)
- 最終チェックポイント(スタート/ゴールライン)を、最初から逆走せずに通過した場合にラップ数を+1する
- ラップ数が3に達してスタート/ゴールラインを通過した時点でそのレーサーは「ゴール」とみなす

### 順位計算

順位は以下の優先順位で決定する。

1. ゴール済みのレーサーは、ゴールした順(早い順)が上位
2. 未ゴールのレーサー同士は「現在のラップ数」が多い方が上位
3. 同ラップの場合は「直近に通過したチェックポイント番号」が大きい方が上位
4. 同チェックポイントの場合は「次のチェックポイントまでの距離が近い方」が上位

この順位は毎フレーム(または一定間隔)で再計算し、Model層の `ReactiveProperty<int>`(順位)としてHUD/CPU AIに公開する。

## レース終了条件

- v1では **プレイヤーがゴールした時点でレース終了** とする(残っているCPUの順位はその時点の暫定順位で確定させる)
  - 理由: 全員がゴールするまで待つ仕様は実装・体験の両面でv1には過剰。プレイヤー体験を優先する
- レース終了後、リザルトシーンへ遷移する([01-scenes.md](01-scenes.md))

## アーキテクチャメモ

- コースのチェックポイント配置・ラップ数などのコース設定は `CourseData` のような `ScriptableObject` として定義する
- レース進行状態(各レーサーの順位・ラップ・チェックポイント)を持つ `RaceModel` を用意し、R3の `ReactiveProperty` / `Observable` で変化を通知する
