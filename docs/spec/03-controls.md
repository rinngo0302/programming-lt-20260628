# 操作仕様

v1ではキーボード操作のみをサポートする(ゲームパッド対応は将来拡張候補)。

## キー割り当て(案)

| 操作 | キー | 説明 |
|---|---|---|
| 加速 | `W` または `↑` | カートを前進させる |
| ブレーキ/バック | `S` または `↓` | 減速、停止後はバック |
| 左ステアリング | `A` または `←` | 左に旋回 |
| 右ステアリング | `D` または `→` | 右に旋回 |
| アイテム使用 | `Space` | 所持しているアイテムを使用する |

ドリフトは含めないため、専用のドリフトキーは割り当てない。

## 既存Input System Actionsとの関係

`Assets/InputSystem_Actions.inputactions` には汎用テンプレートの `Move` / `Look` / `Sprint` / `Jump` などが既に定義されている。これはレースゲーム用ではないため、実装フェーズで以下のように再定義する想定(仕様としてはここまでを決定事項とし、具体的なAction名やバインディングの詳細は実装時に決める)。

- `Move`(Vector2) → ステアリング(X軸)+ 加速/ブレーキ(Y軸)として再利用、または `Steer`(float) / `Accelerate`(float) に分割して新規定義
- アイテム使用用のActionを新規追加する(`UseItem`など)
- レースに不要な `Attack` / `Interact` / `Crouch` / `Previous` / `Next` は削除または未使用のままにする

具体的なAction Map再設計は実装タスクの一部として行う。
